# 参与贡献（Contributing）

感谢你对 GTImaster 的关注与参与。以下说明旨在让协作更顺畅。

---

## 你可以怎样参与

- **反馈问题**：程序报错、界面错位、某分辨率下异常等，欢迎提 Issue。
- **建议功能**：新功能或交互改进，可在 Issue 中描述使用场景与期望行为。
- **提交代码**：修 bug、改文档、实现小功能等，欢迎提 Pull Request（PR）。

---

## 开发环境准备

1. **Fork 本仓库**，然后将你的 Fork 克隆到本地：
   ```bash
   git clone https://github.com/你的用户名/GTImaster.git
   cd GTImaster
   ```

2. **安装依赖**（建议使用虚拟环境）：
   ```bash
   pip install -r requirements.txt
   ```

3. **准备 EasyOCR**：将 `.EasyOCR` 放到用户目录，或使用根目录的 `一键部署.bat` 完成复制与安装。

4. **运行程序**：在项目根目录执行 `python GTImaster.py`，确认能正常启动与操作。

---

## 项目结构简要说明

便于你快速定位要改动的文件：

| 路径 | 说明 |
|------|------|
| `GTImaster.py` | 程序入口，初始化与回调绑定 |
| `config.py` | 界面坐标与全局配置（基准 1920×1080） |
| `core/` | 核心逻辑：常规模式、实时检测、定时关机、状态、快捷键、fast_click 等 |
| `ui/` | 主窗口与两个 Tab 的界面构建 |
| `utils/` | 截图等工具函数 |

- 坐标与分辨率相关逻辑以 **config.py** 和 **core** 内使用 `rel_x/rel_y` 的模块为主。
- 新增依赖请写入 `requirements.txt`，并尽量保持轻量。

---

## 提交 Pull Request 的流程

1. **从本仓库拉取最新代码**，基于 `main`（或当前默认分支）创建你的分支：
   ```bash
   git checkout -b fix/xxx   # 或 feat/xxx、docs/xxx
   ```

2. **在本地修改、自测**，确保：
   - 程序能正常启动；
   - 你改动的功能在 1920×1080（或你测试的分辨率）下行为符合预期。

3. **提交时**：
   - 提交信息尽量简短清晰，如：`修复 2560×1440 下某按钮错位`、`补充 README 分辨率说明`。
   - 若改动涉及配置或行为，可在 PR 描述里说明测试环境（系统、分辨率、缩放）。

4. **推送到你的 Fork**，然后在本仓库页面打开 **Pull Request**，填写标题与描述，说明改动目的与测试情况。

5. **维护者会进行 review**，必要时会请你补充修改，合并后会统一感谢。

## 代码与风格

- **语言**：项目以 Python 为主，注释与文档以中文为主。
- **风格**：保持与现有代码一致即可（如缩进、命名）。无强制格式化工具时，以可读、少改动范围为宜。
- **兼容性**：请兼顾 Windows 10 / 11，以及 README 中说明的 1920×1080、2560×1440、3840×2160 等 16:9 分辨率；若涉及坐标或布局，尽量在 config 或集中配置处修改，便于他人适配。
